# Homepage A 阶段：主题包系统 + 排列优化 — 设计 Spec

**日期**：2026-04-27
**范围**：Sakura Nav 首页主区（导航卡片 + 搜索 + hero + 分组）的视觉与排列优化
**状态**：设计已用户批准，待写实现计划

---

## 1. 背景与目标

Sakura Nav 当前是一个跨 10+ 子系统的个人导航起始页。本 spec 仅覆盖 **A 阶段：首页主区**，是用户优先级 A → C → E → B → F → G → D 序列中的第一个独立循环。

### 用户提出的两个核心需求

1. **排列方便易操作**：高频内容前置，搜索/卡片/分组的访问路径更短，移动端可用
2. **多主题切换**：至少四套预设主题，包含 Q 版二次元风格（**柔和星空、萌系治愈，非高饱和**）

### 不在本 spec 范围内（明确排除，避免范围爆炸）

- `app.js`(172KB) 整体拆分重构 — 将来独立循环
- 性能优化（首屏速度、IndexedDB 大文件加载）— 独立循环
- AI / 日历 / 音乐 / 博客 / 天气 / 同步 子系统的视觉调整 — 各自后续 brainstorm 循环
- 后端、Docker、同步逻辑 — 与首页视觉无关

---

## 2. 整体架构

不重写、不拆模块，**用一个轻量"主题包"层叠加在现有代码之上**，仅在三个关键入口插桩。

```
┌─────────────────────────────────────────────────────┐
│  现有代码（保留）                                      │
│  index.html · app.js (172KB) · 大部分 styles.css     │
└─────────────────────────────────────────────────────┘
                        ↓ 插桩点
┌─────────────────────────────────────────────────────┐
│  入口 1：<html data-theme="…" data-mode="…">         │
│  入口 2：sakura.js 装饰粒子接口（可换渲染器）          │
│  入口 3：app.js 卡片/hero 渲染时附加 theme class      │
└─────────────────────────────────────────────────────┘
                        ↑ 由
┌─────────────────────────────────────────────────────┐
│  新增：theme.js（~300 行）+ themes/*.css（4 个文件）   │
│  · 主题注册表 / 当前主题状态 / 切换逻辑                │
│  · 每套主题：CSS 变量包 + 粒子渲染器 + 字体/卡片配置   │
└─────────────────────────────────────────────────────┘
```

### 文件改动清单

| 文件 | 改动类型 | 估算量 |
|---|---|---|
| `theme.js` | 新增 | ~300 行 |
| `themes/sakura.css` | 新增 | ~100 行 |
| `themes/q-anime.css` | 新增 | ~150 行 |
| `themes/dark.css` | 新增 | ~100 行 |
| `themes/paper.css` | 新增 | ~120 行 |
| `styles.css` | 变量化重构 | 不增行数，替换硬编码值 |
| `sakura.js` | 抽渲染器接口 + 4 个粒子实现 | +200 行（4 个渲染器各 50 行左右） |
| `app.js` | 渲染入口加 hook + hero 合并 + 高频区 + 分组 tab | < 200 行新增/修改 |
| `index.html` | 加载新文件 + 新区块 HTML 骨架 | +30 行 |

## 3. 主题包数据模型

每套主题 = 一个 JS 对象（注册到 `theme.js`）+ 一份 CSS 文件（提供变量值与变体类）。

```js
// theme.js 注册表里每一项
{
  id: 'q-anime',
  name: 'Q 版二次元',
  emoji: '✨',

  cssVars: {
    // 配色（柔和星空版 — 用户明确要求"温和、如星空一般、更萌"）
    '--bg-base':       '#fdf8ff',
    '--bg-glass':      'rgba(255, 240, 250, 0.55)',
    '--accent':        '#c4a8e8',          // 雾紫
    '--accent-2':      '#ffc4d6',          // 樱乳粉
    '--accent-3':      '#a8c8ff',          // 月光蓝
    '--text-strong':   '#4a3a5e',
    '--text-soft':     '#7a6a8e',
    '--card-bg':       'rgba(255, 250, 253, 0.7)',
    '--card-border':   'rgba(196, 168, 232, 0.4)',

    // 形态
    '--card-radius':   '28px',             // Q 版超圆
    '--btn-radius':    '999px',            // 胶囊按钮
    '--card-shadow':   '0 6px 20px rgba(196, 168, 232, .18)',
    '--card-hover-lift': '4px',

    // 字体
    '--font-base':     '"M PLUS Rounded 1c", "PingFang SC", system-ui',
    '--font-display':  '"Fredoka", "M PLUS Rounded 1c", system-ui',
    '--font-size-base':'15px',
    '--letter-spacing':'0.02em',           // 萌系呼吸感

    // 玻璃
    '--glass-blur':    '14px',
    '--glass-saturate':'1.4',
  },

  particles: {
    renderer: 'candy-stars',
    count: 50,
    speed: 0.6,
    palette: ['#c4a8e8', '#ffc4d6', '#a8c8ff', '#fff4a8'],
  },

  cardVariant: 'q-anime',                  // app.js 渲染时附加 .card--q-anime 类
  heroStyle: 'compact',
  enterAnim: 'bounce',
}
```

### 四套预设差异

| 维度 | 🌸 樱粉 | ✨ Q 版二次元 | 🌙 暗夜极简 | 📜 复古纸质 |
|---|---|---|---|---|
| **调性** | 现行风格 | 柔和星空 / 萌系治愈 | 静默高对比 | 怀旧温润 |
| **粒子** | sakura（樱花） | candy-stars（小星星 + 流星 + 极少量小云） | none | paper-leaves（纸屑 + 落叶） |
| **主色** | 樱粉 #ff8ab1 | 雾紫 #c4a8e8 + 樱乳粉 + 月光蓝 三色低饱和 | 月光蓝灰 #8da4c0 | 墨绿 #4a5b3e + 赭石 #b07c4f |
| **圆角** | 16px | 28px | 8px | 6px |
| **卡片** | 玻璃模糊 | 软糖卡：高圆角 + 柔阴影 + 半透明渐变 | 极简描边 | 纸质阴影 + 米黄底 |
| **字体** | 默认无衬线 | 圆润字 + 字间距 | 等宽 / 极简 | 衬线 / 手写 |
| **悬停** | 轻微上浮 | 微微上浮 + 边缘星光 | 描边亮起 | 抬起 + 纸感阴影 |
| **暗色版** | ✓ | ✓（深紫宇宙底 + 银河淡光） | 本身即暗 | ✓（夜读纸） |

### 关键决策

- **主题对象是数据，不是代码** — 加新主题 = 写一个对象 + 一份 CSS，不改任何逻辑
- **现有的"主色调微调"功能保留** — 作用于当前主题的 `--accent` override，存独立 key (`accentOverride`)，切主题时自动应用
- **暗色 / 亮色 / 跟随系统是主题之上的二级开关** — 每套主题各自实现 `[data-theme="x"][data-mode="dark"]` 选择器

## 4. 布局改动

### 4.1 hero 区合并

**现状**：时钟 / 问候 / 一言 三块各占一行，共 ~140px 高
**改后**：

```
┌────────────────────────────────────────────────────┐
│  下午好，muxin · 14:32                              │
│  "万物皆有裂痕，那是光照进来的地方。" — 一言        │
└────────────────────────────────────────────────────┘
```

- 一行问候+时钟，下一行一言；总高 ~70px
- 一言交互保留：点击刷新 / 长按隐藏
- 设置中三档选择：**展开 / 紧凑（默认）/ 隐藏**

### 4.2 高频区前置（默认开）

紧跟 hero 区下方、搜索栏上方插入：

```
[ ⭐ 星标置顶（横向滚动） ]
[ 🕒 最近使用（10 个，横向滚动） ]
```

- 首屏不滚动可见，移动端尤其受益
- 设置可关
- 数据来自现有的 `clickCount` / `lastClickAt` / 星标字段

### 4.3 粘性分组 tab

分组数 ≥ 4 时自动启用：

```
[全部] [工作] [学习] [娱乐] [开发] [资源]    ← 粘性置顶
   ↓
具体卡片网格...
```

- ≤ 3 组时不显示，避免冗余
- 点击 = 平滑滚动到该组 + 高亮当前
- 桌面端吸顶；**移动端也吸顶**（不与系统手势/键盘冲突）
- 横向滚动条隐藏，使用渐变 mask 提示可滑

### 4.4 移动端响应式（≤768px）

- 樱花 / 任意主题粒子数 ÷ 2，玻璃模糊降一档
- 卡片网格：单列 → **两列**（紧凑档三列）
- hero 区折叠为单行
- 设置面板从 modal 改为底部抽屉
- 分组 tab 条吸顶

### 4.5 卡片密度新增"超紧凑"档

四档：紧凑 / 常规 / 宽松 / **超紧凑（图标墙）**
超紧凑档：仅 favicon + 缩略名称，每行 6-8 个，适合"我就是要快速跳"。

## 5. 粒子渲染器接口

`sakura.js` 的"粒子是樱花"硬编码改为插件式：

```js
// sakura.js 顶部新增的注册表 API
ParticleEngine.register('sakura', {
  init(ctx, config)  { /* 初始化粒子数组 */ },
  step(ctx, dt)      { /* 每帧推进 + 绘制 */ },
  resize(w, h)       { /* 画布尺寸变更 */ },
  destroy()          { /* 清理 */ },
});
ParticleEngine.register('candy-stars',  { /* Q 版：星星 + 偶发流星 + 极少小云 */ });
ParticleEngine.register('paper-leaves', { /* 纸屑 + 落叶 */ });
ParticleEngine.register('none',         { /* 空实现 */ });

// 主题切换时
ParticleEngine.use('candy-stars', themeConfig.particles);
```

### 公共逻辑保留在引擎层

- `<canvas>` 创建 / 销毁 / resize
- requestAnimationFrame 调度
- 页面隐藏自动暂停
- `prefers-reduced-motion` → 0 粒子
- 移动端粒子数 ÷ 2

### 各渲染器风格

- **sakura**：现有飘樱花，逻辑搬入即可，行为不变
- **candy-stars**：背景层缓慢飘 ✦ 小星 + 极少量 ☁ 小云 + 30 秒一次的流星划过；调色板由主题提供
- **none**：空实现，立即返回（暗夜极简使用）
- **paper-leaves**：飘纸屑 + 落叶，赭/墨/米黄色调，下落速度比樱花略慢

## 6. 持久化与切换机制

### 6.1 存储

新增 localStorage key：

```js
localStorage["sakura_nav_theme_v1"] = {
  themeId: "q-anime",
  mode: "auto",                    // auto | light | dark
  accentOverride: null,            // 用户微调的主色（hex），null = 用主题默认
  particleEnabled: true,
  particleCount: null,             // null = 用主题默认
}
```

服务端模式下打包进现有的 `app_data` JSON 同步。

### 6.2 切换流程

```
用户在设置里点"复古纸质"
   ↓
1. theme.js 读出主题对象
2. <html data-theme="paper" data-mode="light">
3. 对应 themes/paper.css 已在启动时预加载，规则即刻生效
4. ParticleEngine.use('paper-leaves', config)
5. 触发 enterAnim（淡入 / 弹入 / 滑入）
6. localStorage 写入
7. 派发 'theme:changed' 事件，AI/日历/音乐等子系统监听做联动（如有需要）
```

**热切换不刷页**：所有 4 套主题 CSS 在启动时都已 `<link rel="stylesheet">` 加载，但只有匹配 `[data-theme="x"]` 的规则生效，切换 = 改属性即可。

### 6.3 设置面板 UI

在现有设置中新增"主题"区块：

- 4 张主题预览卡（缩略图 + 名称 + 当前选中徽章）
- 主色微调（沿用现有调色板，作用于当前主题的 `--accent` override）
- 模式：跟随系统 / 亮色 / 暗色（三选一）
- 粒子：开关 + 数量（保留现有）

## 7. 错误处理

| 场景 | 行为 |
|---|---|
| 主题 ID 不存在（旧版本残留 / 损坏数据） | 回退到 `sakura` 默认主题，控制台 warn |
| 粒子渲染器报错 | 引擎自动切到 `none`，不影响布局 |
| Canvas 不可用（旧浏览器） | 静默禁用粒子，主题颜色照样生效 |
| `localStorage` 写失败（隐私模式 / 配额） | 切换在内存生效，刷新后回到默认 |
| 移动端粒子卡顿（FPS < 30 检测） | 自动减半粒子数；连续 3 次仍卡，切到 `none` 并 toast 提示 |

## 8. 测试策略

项目无现成测试框架，采用三层手工 + 一层冒烟。

### 8.1 视觉手测清单（每套主题 × 6 场景）

每套主题在以下场景截图：
1. 首页默认（hero + 高频区 + 分组卡片）
2. 卡片悬停态
3. 设置面板打开
4. AI 助手面板打开
5. 日历视图
6. 移动端 ≤768px

共 4 主题 × 6 场景 = 24 张截图归档。

### 8.2 交互手测清单

- [ ] 切换主题不刷页
- [ ] 主色微调生效，切回另一主题再切回，微调保留
- [ ] `prefers-reduced-motion` 启用时所有主题粒子为 0
- [ ] 粒子开关关闭后任何主题都无粒子
- [ ] 移动端 ≤768px 布局正确（两列卡片、hero 单行、tab 吸顶、设置抽屉）
- [ ] 暗色 / 亮色 / 跟随系统三种模式 × 4 主题 共 12 组合不破图
- [ ] 高频区 ⭐ 和 🕒 数据正确（最近 10 个、星标置顶卡）
- [ ] 分组 ≥ 4 时 tab 条出现，≤3 时不出现
- [ ] tab 点击平滑滚动并高亮

### 8.3 回归手测

现有功能在四套主题下不破：
- 拖拽（卡片跨组、整组重排）
- 添加链接（粘贴 URL 弹窗、表单提交）
- 搜索（输入、引擎切换、联想下拉）
- AI 唤起（`Alt+A`、面板渲染、流式回复）
- 设置面板既有项（樱花、玻璃、背景图、账号）

### 8.4 冒烟自动

在现有 `_smoke.ps1` 中新增 grep 检查：

```powershell
# 主题系统就位
Test-Path "themes/q-anime.css"
Select-String -Path "theme.js" -Pattern "ParticleEngine.register"
Select-String -Path "index.html" -Pattern 'data-theme'
# 4 套主题都注册了
$themes = @('sakura', 'q-anime', 'dark', 'paper')
foreach ($t in $themes) { Select-String "theme.js" -Pattern "id:\s*'$t'" }
```

## 9. 验收标准（Definition of Done）

满足全部以下条件方可结束 A 阶段：

1. 4 套主题在桌面 + 移动端均可切换、所有 6 个场景视觉无破图
2. hero 紧凑 / 高频区 / 粘性分组 tab / 移动端响应式 / 超紧凑卡片密度 全部可用
3. 现有功能（README §组件 & 交互 列出的所有项）回归测试 100% 通过
4. 切主题时不刷页、主色微调持久化、暗色模式正确
5. 冒烟脚本通过
6. README 更新主题相关说明
